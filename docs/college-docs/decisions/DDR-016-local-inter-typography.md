# DDR-016: локальний шрифт Inter як основа типографіки сайту

## Статус
Прийнято

## Дата
2026-06-30

## Контекст
Під час аудиту типографіки було перевірено, які шрифти реально використовує демоверсія сайту. У публічному CSS був один основний стек: `Inter`, системні sans-serif fallback-шрифти і загальна категорія `sans-serif`. Окремих підключень Google Fonts, `@font-face` або локальних файлів шрифтів для сайту не було.

Це означало, що назва `Inter` була вказана в CSS, але фактичний вигляд залежав від пристрою користувача. Якщо Inter не встановлено локально, браузер переходив до системного шрифту. Для сайту коледжу важливо мати послідовну, читабельну і юридично безпечну типографіку без зайвої кількості гарнітур.

## Рішення
- Основною гарнітурою сайту прийнято **Inter**.
- Базовий CSS-стек сайту: `Inter, system-ui, sans-serif`.
- Inter підключається локально з репозиторію, а не через зовнішній CDN або Google Fonts.
- У демоверсії додано файли:
  - `assets/fonts/inter/InterVariable.woff2`;
  - `assets/fonts/inter/InterVariable-Italic.woff2`;
  - `assets/fonts/inter/OFL.txt`.
- Використовується Inter v4.1 з офіційного репозиторію `rsms/inter`.
- Ліцензія Inter - **SIL Open Font License**. Шрифт можна використовувати на сайті без купівлі окремої комерційної ліцензії.
- `system-ui` і `sans-serif` залишаються тільки fallback-варіантами на випадок, якщо локальний файл шрифту не завантажиться.
- Нові зовнішні шрифти, Google Fonts-підключення або платні/proprietary гарнітури не додавати без окремого дизайн-рішення.

## Альтернативи
- **Залишити тільки системні шрифти**: відхилено, бо вигляд сайту помітно різнився б між Windows, macOS, Android і iOS.
- **Підключити Inter через Google Fonts або інший CDN**: відхилено для базової реалізації, бо локальний файл дає стабільніший контроль, не створює зовнішньої залежності й краще підходить для майбутньої production-передачі.
- **Використовувати кілька різних гарнітур для заголовків і тексту**: відхилено на цьому етапі, бо сайт коледжу має виглядати стримано, інституційно і послідовно. Різноманіття має створюватися ієрархією, вагою, розміром, кольором і відступами, а не кількістю шрифтів.
- **Використати системні proprietary-шрифти на кшталт Segoe UI або San Francisco як основні webfont-файли**: відхилено, бо їх не варто розповсюджувати як файли сайту без окремої ліцензійної перевірки.

## Наслідки
- Усі сторінки демоверсії мають використовувати єдину гарнітуру Inter через глобальний CSS.
- Типографічна система сайту має будуватися на одній гарнітурі: різні ролі тексту задаються розміром, вагою, line-height, кольором і простором.
- Майбутня WordPress-реалізація має постачати Inter локально в темі або у погодженому asset-пакеті, а не покладатися на те, що шрифт встановлений у користувача.
- Файл ліцензії потрібно зберегти поруч із файлами шрифту або в іншому зрозумілому місці production-репозиторію.
- Під час review потрібно перевіряти, що файл `InterVariable.woff2` завантажується з локального шляху, а сайт не робить запитів до зовнішніх font-сервісів.
- Якщо в майбутньому буде запропоновано другу гарнітуру, потрібно окремо оцінити її кирилицю, ліцензію, вагу файлів, вплив на швидкість завантаження і відповідність інституційному стилю коледжу.
