# DDR-012: тема оформлення день/ніч і прибирання демо-перемикача тем

## Статус
Прийнято

## Дата
2026-06-30

## Уточнення
2026-07-01 після обговорення з колегами й адміністрацією коледжу підтверджено фінальну модель для демоверсії: залишаються тільки основна денна тема і нічний режим, а перемикання відбувається через іконку в header одним кліком без випадаючої панелі.

## Контекст
У демоверсії був плаваючий демо-перемикач тем у правому нижньому куті екрана з чотирма напрямами кольорів: основна, нічна, фіолетова і бордова. Він був інструментом для попереднього перегляду кольорових напрямів під час review, а не елементом інтерфейсу для кінцевого користувача.

Для production-сайту потрібен зрозумілий, передбачуваний перемикач світлої та темної теми безпосередньо в header, а не плаваючий службовий блок із кількома експериментальними палітрами.

## Рішення
- У header додається кнопка-перемикач теми день/ніч з іконкою сонця/місяця. Вона показується разом з іншими верхніми діями (пошук, мова) на всіх адаптивних розкладках.
- Підтримуються рівно дві теми: основна (день) і нічна (ніч). Інші демо-теми (фіолетова, бордова) прибираються разом із плаваючим демо-перемикачем.
- Перемикач працює як прямий toggle: один клік вмикає нічну тему, наступний клік повертає основну денну тему. Окрема панель вибору тем не використовується.
- Нічна тема застосовується через атрибут `data-theme="night"` на кореневому елементі; день - це відсутність атрибута.
- Вибір теми зберігається локально (ключ `site-theme`) і відновлюється під час наступних завантажень. Невідоме або застаріле збережене значення трактується як день.
- Іконка перемикача відображає поточний стан: місяць у денному режимі (клік вмикає ніч), сонце в нічному режимі (клік вмикає день). Підпис `aria-label` і `title` локалізовані й описують дію перемикання.
- Усі кольори теми беруться із системних CSS-змінних. Нічна тема перевизначає той самий набір змінних, тому окремі компоненти не потребують власних кольорів під ніч.

## Альтернативи
- **Залишити плаваючий демо-перемикач із чотирма темами**: відхилено, бо це службовий інструмент review, а не елемент production-інтерфейсу.
- **Орієнтуватися лише на системний `prefers-color-scheme` без кнопки**: відхилено для базової версії, бо користувач має мати явний контроль над темою в header. Системну преференцію можна врахувати додатково в майбутньому.
- **Реалізувати багатотемність із вибором палітри**: відхилено, бо для коледжу достатньо зрозумілої пари день/ніч без зайвого вибору.

## Наслідки
- У демоверсії перемикач день/ніч додається через спільний скрипт `js/main.js` і не дублюється вручну в кожному HTML-файлі.
- CSS зберігає лише змінні основної та нічної теми; правила для прибраних тем видалені.
- У майбутній CMS / production-реалізації перемикач теми має бути частиною глобального header, а збереження вибору - через локальне сховище або погоджений механізм теми.
- Під час review потрібно перевіряти, що в header є перемикач день/ніч, один клік перемикає тему, нічна тема застосовується до всіх ключових компонентів, а плаваючий демо-перемикач і додаткові теми відсутні.
