# DDR-006: часткова англомовна модель сайту

## Статус
Прийнято

## Дата
2026-06-29

## Уточнення
2026-07-01 підтверджено, що часткова англомовна модель потрібна саме для зменшення редакційного навантаження. Динамічний контент має створюватися один раз українською мовою в майбутній CMS, без обов’язкового дублювання для англійської версії.

## Контекст
Для сайту коледжу потрібна англійська присутність, але немає редакційного ресурсу для повного паралельного ведення всього контенту двома мовами. Повна двомовність у CMS означала б, що новини, оголошення, документи, медіа й частина сторінок довелося б дублювати або перекладати окремо. Це створює постійне адміністративне навантаження і ризик розсинхрону між мовними версіями.

Водночас англійський інтерфейс потрібен для базової зрозумілості сайту: глобальна навігація, header, footer, службові сторінки й ключові статичні сторінки мають виглядати як підтримана англомовна версія.

## Рішення
- Українська мова є основною мовою сайту.
- Англійська підтримується як часткова мовна версія: англомовний інтерфейс, глобальні компоненти, службові елементи та окремі статичні сторінки.
- Головна сторінка має мати англійську версію статичного контенту. Новини на англомовній головній сторінці можуть залишатися українською.
- Окремі статичні сторінки розділів, зокрема «Коледж», «Абітурієнту», «Студенту», «Випускнику», «Наука», «Бібліотека», мають бути перекладені англійською.
- Динамічний контент майбутньої CMS, зокрема новини, оголошення, документи, матеріали розділів і медіа, за замовчуванням публікується один раз українською мовою.
- Для динамічного контенту не створювати обов'язковий англійський дубль запису, сторінки або медіафайлу.
- На англомовних сторінках із динамічним українським контентом перекладаються header, footer, кнопки та службові елементи інтерфейсу; сам контент може залишатися українською.
- Перемикач мови лишається коротким: «UA» і «EN», без додаткового пояснювального тексту в інтерфейсі.

## Альтернативи
- **Повна двомовність усього контенту**: відхилено, бо це подвоює редакційне навантаження і змушує дублювати матеріали, які фактично не перекладаються.
- **Лише українська версія без англійського інтерфейсу**: відхилено, бо сайту потрібна базова англійська присутність.
- **Автоматичний машинний переклад усього сайту**: не прийнято як базову вимогу. Такий підхід можна окремо обговорити пізніше, але він не має бути умовою поточної CMS / production-реалізації.
- **Формальні англійські копії сторінок із тим самим українським контентом**: відхилено, бо це не зменшує редакційне навантаження і створює ризик розсинхрону.

## Наслідки
- Демоверсія має симулювати саме цю модель: перекладені глобальні елементи й окремі статичні сторінки, але без вимоги перекладати весь контент.
- Майбутня CMS / production-реалізація має локалізувати інтерфейсні рядки: header, footer, меню, пошук, CTA, службові повідомлення, 404 та інші елементи теми.
- Редактор CMS має мати простий базовий сценарій: додати новину або оголошення один раз українською, без обов'язкового створення англійської копії.
- Англійські версії потрібно готувати тільки для погоджених статичних сторінок і глобальних елементів, якщо не буде окремого рішення про ширший переклад.
- Медіафайли не повинні фізично дублюватися тільки через мовну версію.

## Перевірка
- На англомовній головній сторінці статичний контент має бути англійською, а блок новин може показувати українські новини.
- Header, footer, пошук, CTA, 404 і службові повідомлення мають мати англійські тексти.
- Додавання української новини у CMS не повинно вимагати створення англійського дубля для того, щоб матеріал міг відображатися в англомовному інтерфейсі.
- Перемикач мови має вести на погоджені англійські сторінки там, де вони існують, і не створювати порожні сторінки лише заради формальної двомовності.
